# Important Constraints

### Slug-based routing

Frontend must **never use product IDs**.

Always use:

/products/:slug

Example backend endpoint:

GET /catalog/products/slug/:slug

IDs remain backend-only.

---

### Support both users

The system supports:

1. Guest users
2. Logged-in users

Guest flows:

* cart stored locally
* checkout uses guest order endpoint

Logged-in flows:

* cart synced with backend
* wishlist available
* addresses saved
* orders saved

Homepage Product Sections

Trending
Best Sellers
Recommended
Deals
New Arrivals
Highly Rated
Most Wishlisted
Also Viewed

# Cart Page

Cart works differently depending on user.

Guest users:

cart stored locally.

Logged-in users:

GET /me/cart
POST /me/cart/items
DELETE /me/cart/items/:id

Cart page must allow:

place order
buy via WhatsApp

# Checkout Page

Flow:

Cart Summary
Shipping Address
Payment
Place Order

Guest checkout:

POST /orders/guest

Logged-in checkout:

POST /orders

Shipping info rules:

Guest → provide shipping info.

Logged-in → choose from saved addresses.

Address endpoints:

GET /me/addresses
POST /me/addresses
PUT /me/addresses/:id

Payment options:

Cash on delivery (enabled)
